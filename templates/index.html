Presentation guide

            
Class Defense Talking Points

          

        

        

          

            
1. Problem

            
Students and small creators waste time switching between writing tools, planners, and social platforms. They need one simple workspace for content, school help, and scheduling.

          

          

            
2. Solution

            
SocialPilot AI combines login, AI content generation, drafts, scheduling, a calendar, and a student assistant in one dashboard.

          

          

            
3. Target Users

            
Students, school clubs, small businesses, content creators, and teams that need help planning and publishing content.

          

          

            
4. Core MVP

            
Landing page, Supabase login, dashboard, AI generator, drafts, calendar, assistant, accounts, team roles, tasks, and admin settings.

          

          

            
5. Architecture

            
Python backend, HTML/CSS/JavaScript frontend, Supabase Auth, OpenAI/Gemini AI provider support, and local JSON storage for MVP data.

          

          

            
6. Security

            
Normal users do not see API keys. Admin-only settings store AI keys server-side. Social posting uses official APIs only.

          

          

            
7. Real Data Policy

            
The app does not fake likes, reach, or comments. It only shows real local activity and marks external analytics as requiring official API permissions.

          

          

            
8. Limitations

            
Auto-posting and analytics need approved Instagram/X API credentials. Always-on scheduling needs cloud hosting or another always-on server.

          

          

            
9. Team Contribution

            
Roles can be split into frontend, backend, AI prompts, Supabase auth, social API research, testing, documentation, and presentation.
